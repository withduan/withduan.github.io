realme Q3s & realmeUI 2.0 系统 App 精简列表

2025/02/11 教程 Android realme realmeUI 真我

本文中系统 App 列表来自 realme Q3s（RMX3471）。将手机恢复出厂设置后，第一次开机后使用第三方工具导出已安装 App 列表，内容应该非常齐全（仅针对我使用的固件版本：RMX3461_11_A.13）。

预装第三方 App（如微博、京东、饿了么等）不再列出，其余 App 按 package name 排序，方便检索，顺便增加了对应的 adb 操作和还原指令。

根据我自己的使用需求，把手机里系统云服务、查找手机等等核心功能都删掉了。主力机极端精简后可能会影响日常使用，请根据自身情况进行取舍，内容仅供参考。

备注： 如果出现 adb 指令无法删除某个 App 的情况，可以尝试先清除 App 数据后再操作，如果还是无法卸载说明该 App 不支持卸载，可以尝试冻结。

一、macOS 无线 adb 调试

Q3s 这台手机不知道有什么毛病，连接到 MacBook 时，手机反复识别又断开， 症状就像接触不良一样，换了 C2C、A2C 数据线问题依旧（但是连接 Windows 设备又一切正常），所以用不了 USB 调试，只能尝试通过 WiFi 无线调试。

1.通过 Homebrew 安装 adb

我的电脑安装过 Homebrew，直接在终端执行brew install --cask android-platform-tools

系统没有 Homebrew 的话先执行该指令安装 Homebrew： /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

安装结束之后，在终端执行指令：adb version，会出现 adb 版本信息。

2.使用 adb 连接 Android 设备

首先确保 Android 设备和 Mac 连接到了同一个局域网。

打开 Q3s 的 设置 - 其他设置 - 开发者选项，找到「无线调试」，打开该选项右侧的开关。

「无线调试」同时也是一个可点击的项目（究竟是哪个大聪明发明的交互），点击该选项，再点击「使用配对码配对设备」。

此时会弹出一个提示框，显示当前的配对码（6 位数字）、IP 地址、端口。

在 Mac 终端执行 adb pair <IP:端口>，假设刚才 Android 设备显示配对的 IP 地址是 192.168.1.100，端口 123456，这里需要执行的指令是： adb pair 192.168.1.100:12345，再根据提示输入 6 位配对码便可配对成功。

现在只是配对成功了，并没有连接上，还得执行adb connect <IP:端口>再进行连接。此时使用的端口号不是前面配对用的端口号，刚才的端口号只能用在配对流程中，每次配对新设备时都会刷新端口号和配对码。这里需要使用调试端口号，在「无线调试」设置内可以找到。无线调试功能开关变动时，此处的端口号也会刷新。

假设此处的调试端口号为 23333，在终端执行`adb connect 192.168.1.100:23333

不出意外的话此时已经连接成功了，手机通知栏会跳出无线调试的通知。

3. 使用 adb 指令清理无用系统 App

连接后可以在 macOS 终端使用以下指令卸载、冻结、恢复指定 App

卸载应用：adb shell pm uninstall --user 0 <package name>

卸载恢复：adb shell pm install-existing <package name>

冻结应用：adb shell pm suspend <package name>

冻结恢复：adb shell pm unsuspend <package_name>
